# 학교생활기록부 점검 스킬

## 개요

중학교 학교생활기록부(학생부) PDF를 체계적으로 분석하여 기재요령 위반, 오탈자, 띄어쓰기, 문장 미완성, 의미불명, 부적절 표현 등을 점검하고 마크다운 보고서를 생성한다.

## 사용법

```
/check-hakbu <PDF 파일 경로>
/check-hakbu            # 인자 없으면 사용자에게 경로 질문
```

복수 파일 점검 시 파일별로 반복 호출하거나, 폴더 경로를 지정하여 일괄 처리한다.

---

## 점검 워크플로우

### Phase 0: 입력 확인

1. 인자로 전달된 PDF 경로를 확인한다. 없으면 사용자에게 질문한다.
2. PDF 파일 존재 여부를 확인한다.
3. 출력 보고서 경로를 결정한다: `{PDF파일명}_검토보고서.md` (동일 디렉터리)

### Phase 1: PDF 전처리 (Python)

```python
import pikepdf, fitz, os

pdf_path = "<입력 PDF>"
base_dir = os.path.dirname(pdf_path)
base_name = os.path.splitext(os.path.basename(pdf_path))[0]

# 1) 암호 해제 (pikepdf)
try:
    pdf = pikepdf.open(pdf_path)
except Exception:
    pdf = pikepdf.open(pdf_path, password='')
unlocked_path = os.path.join(base_dir, f"{base_name}_unlocked.pdf")
pdf.save(unlocked_path)
pdf.close()

# 2) 페이지별 이미지 변환 (PyMuPDF, dpi=200)
doc = fitz.open(unlocked_path)
page_paths = []
for i, page in enumerate(doc):
    img_path = os.path.join(base_dir, f"_hakbu_p{i+1:02d}.png")
    pix = page.get_pixmap(dpi=200)
    pix.save(img_path)
    page_paths.append(img_path)
total_pages = len(doc)
doc.close()
```

### Phase 2: 이미지 읽기 (Read 도구)

- 전체 페이지 이미지를 Read 도구로 읽는다 (한 번에 6장씩, 병렬 호출).
- 읽은 내용을 바탕으로 학생 기본정보를 파악한다:
  - 학교명, 학생명, 학년/반/번호, 담임, 출력일
  - 총 페이지 수, 학년 범위

### Phase 3: 병렬 분석 (5개 에이전트 팀)

Task 도구를 사용하여 **5개 에이전트를 동시 실행**한다. 각 에이전트에게 해당 페이지 이미지 경로를 전달하여 직접 이미지를 읽게 한다.

#### 팀 1: 기본정보/형식/출결/수상/학교폭력
- **담당 페이지**: 1페이지
- **검토 항목**:
  - 인적사항: 성명, 성별, 주민등록번호(13자리, 검증번호), 주소(도로명주소 형식)
  - 학적사항: 졸업일/입학일 논리적 정합성, 학령 적정성
  - 출결상황: 수업일수(190일 이상), 개근 표기, 빈칸 표기(0/./공란)
  - 수상경력: "해당 사항 없음" 띄어쓰기
  - 학교폭력: "해당 사항 없음" 띄어쓰기
  - 담임성명 기재 여부
- **출력**: `{base_dir}/_analysis_part1.md`

#### 팀 2: 창의적 체험활동상황
- **담당 페이지**: 창체 해당 페이지들 (보통 1~3페이지)
- **검토 항목**:
  - 자율활동: 임원기간 표기(yyyy.mm.dd.~yyyy.mm.dd.), 활동내용 구체성, 문장 완결성
  - 동아리활동: 동아리명 괄호 표기, 학교스포츠클럽 형식, 자율동아리 형식
  - 진로활동: 희망분야(2학년~), 활동내용 구체성, 문장 완결성
  - 시간 배분 적정성 (연간 102~136시간)
  - 종결어미(~함/~임/~음) 사용
  - 부정적 서술만 있는지 확인 (성장 중심 기재 원칙)
  - 날짜 표기 통일성
- **출력**: `{base_dir}/_analysis_part2.md`

#### 팀 3: 교과학습발달상황 (하위 학년)
- **담당 페이지**: 성적표 + 세특 (하위 학년 해당)
- **검토 항목**:
  - 성적표: 자유학기 P 표기, 원점수/과목평균 형식, 성취도(수강자수)
  - 수강자수 정합성 (모든 과목 동일한지)
  - 세특: 문장 완결성, 오탈자, 의미불명, 과목명 혼동
  - 성취도와 세특 내용 정합성 (E등급인데 우수 기재 등)
  - 체육/예술 성취도 (A/B/C 3단계)
  - 교양교과 이수시간/이수여부
- **출력**: `{base_dir}/_analysis_part3.md`

#### 팀 4: 교과학습발달상황 (상위 학년)
- **담당 페이지**: 성적표 + 세특 (상위 학년 해당)
- **검토 항목**: 팀 3과 동일 기준
- **출력**: `{base_dir}/_analysis_part4.md`

#### 팀 5: 봉사활동/자유학기활동/독서활동/행동특성
- **담당 페이지**: 해당 페이지들
- **검토 항목**:
  - 봉사활동: 날짜 형식, 장소/주관기관명, 시간 누계 정확성, 학년별 초기화
  - 자유학기활동: 4영역(진로탐색/주제선택/예술체육/동아리), 시간합계(170시간 내외), 활동명/시간 표기
  - 독서활동: "(학기) 도서명(저자명)" 형식, 학년별/과목별 기재, 누락 여부
  - 행동특성: 학년별 기재, 긍정적 표현 위주, 구체적 사례, 종결어미, 부정적 표현
- **출력**: `{base_dir}/_analysis_part5.md`

## 점검 기준 상세 (학생부 기재요령 근거)

### 공통 기준
- 종결어미: "~함", "~임", "~음", "~됨" 사용 (명사형 종결 불가)
- "해당 사항 없음" (띄어쓰기 포함, "해당사항" 아님)
- 날짜: "yyyy.mm.dd." 또는 "yyyy.mm.dd.~yyyy.mm.dd." (물결표 사용, 하이픈 불가)
- 긍정적·성장 중심 기술 원칙 (부정적 서술만 불가)
- 학생 개인의 특성이 구체적으로 드러나야 함

### 인적·학적사항
- 주민등록번호: 13자리 (6-7), 검증번호 알고리즘 확인
- 주소: 도로명주소 형식
- 학적: 졸업일→입학일 논리적 정합 (통상 1~2월 졸업, 3월 입학)

### 출결상황
- 수업일수: 중학교 190일 이상 (초중등교육법 시행령 제45조)
- 개근: 결석·지각·조퇴·결과 모두 0이면 특기사항에 "개근" 기재
- 빈칸: "0" 또는 "." 또는 공란 (교육청별 상이)

### 창의적 체험활동상황
- 자율활동: 임원기간 yyyy.mm.dd.~yyyy.mm.dd. 형식
- 동아리: "(동아리명)(시간)", "(종목: 학교스포츠클럽)(시간)", "(동아리명: 자율동아리)"
- 진로: 희망분야는 중2부터 기재 (중1은 미기재)
- 연간 시간: 통상 102~136시간

### 교과학습발달상황
- 자유학기: 성취도 P(Pass), 원점수/과목평균 공란
- 일반학기: 원점수/과목평균, 성취도(수강자수) 표기
- 체육/예술: 성취도만(A/B/C), 원점수 없음
- 교양교과: 이수시간, 이수여부(P)
- 세특: 모든 학생 모든 과목 기재 원칙 (2024학년도~)

### 봉사활동실적
- 날짜: yyyy.mm.dd.
- 장소: "(학교)학교명" 또는 "(개인)기관명"
- 누계시간: 학년별 초기화

### 자유학기활동상황
- 4영역: 진로탐색, 주제선택, 예술체육, 동아리
- 시간 합계: 통상 170시간 내외 (17주 x 10시간)
- "(활동명)(시간)" 형식

### 독서활동상황
- 형식: "(학기) 도서명(저자명)"
- 동일 학기 복수 도서: 쉼표(,) 구분
- 학년-과목별 기재

### 행동특성 및 종합의견
- 학년별 기재 필수
- 긍정적 표현 위주 (부정적 표현 순화)
- 구체적 행동 사례 포함
- 종결어미 통일

## 주의사항

1. **이미지 기반 PDF**: 학생부 PDF는 대부분 이미지 기반(스캔)이므로 텍스트 추출이 아닌 이미지 변환 후 Read 도구로 직접 읽는다
2. **암호 보호**: pikepdf로 빈 암호 또는 무암호 해제 시도. 실패하면 사용자에게 암호 요청
3. **OCR 한계**: 이미지 해상도/인쇄 품질에 따라 오독 가능. 보고서에 "NEIS 원본 대조 권장" 명시
4. **개인정보**: 주민등록번호 등 민감정보가 포함되므로 임시 파일은 반드시 삭제
5. **Python 실행**: `python` 명령 사용 (python3 아님). pikepdf, PyMuPDF 설치 필요
6. **에이전트 병렬**: 5개 에이전트를 반드시 동시(병렬) 실행하여 속도를 극대화한다
7. **cp949 인코딩**: Windows 환경에서 print() 시 유니코드 특수문자 주의
