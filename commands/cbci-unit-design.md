# 전북형 개념기반 탐구학습 단원설계 (CBCI Unit Design)

## 개요

교과서 또는 학습자료를 기반으로 전북형 개념기반 탐구학습 단원설계 템플릿을 완성한다.
1.pdf(개념기반수학 철학), 2.pdf(전북형 템플릿), 3.pdf(탐구기법)의 프레임워크를 기반으로
발견 중심형 / 원리 탐구형 / 과정 구성형 중 적절한 모형을 선택하여 설계한다.

## 사용법

```
/cbci-unit-design <단원명> [학년] [자료 경로]
/cbci-unit-design 유리수와 순환소수 중2
/cbci-unit-design 일차함수 중2 ./교과서.pdf
/cbci-unit-design 도형의 닮음 중3 ./자료폴더/
/cbci-unit-design            # 인자 없으면 사용자에게 질문
```

### 지원 파일 형식

| 형식 | 확장자 | 처리 방법 |
|------|--------|----------|
| PDF | `.pdf` | Read 도구로 직접 읽기 (암호 보호 시 pikepdf 해제) |
| 텍스트 | `.txt`, `.md` | Read 도구로 직접 읽기 |
| DOCX | `.docx` | Read 도구로 직접 읽기 |
| 이미지 | `.png`, `.jpg` | Read 도구로 시각적 판독 (교과서 스캔본 등) |
| 디렉터리 | 폴더 경로 | Glob으로 내부 파일 탐색 후 순차 읽기 |

### 파일 소스 처리 흐름

1. **인자에 경로가 있으면** → 해당 파일/폴더를 Phase 1에서 Read로 읽고 교육과정 분석에 활용
2. **경로 없이 단원명만 제공** → AI의 교육과정 지식으로 설계 (사용자에게 고지)
3. **여러 파일 제공** → 디렉터리 경로로 묶어서 제공하면 내부 파일 자동 탐색

### Phase 2: 설계 의사결정 (사용자 확인)

AskUserQuestion으로 핵심 설계 요소를 확인한다:

**질문 1: 핵심개념(렌즈) 선정**
- framework_reference.md의 26개 핵심개념 중 2~3개 제안
- 단원 특성에 맞는 개념을 분석하여 추천 (첫 번째 옵션에 "(Recommended)" 표시)
- 선택지 예: "패턴 + 관계", "패턴 + 관계 + 형태", "구조 + 변화" 등

**질문 2: 총 차시 수**
- 단원 분량에 따라 8~16차시 범위에서 제안
- 소주제 2~4개 × 소주제당 4~5차시로 구성

**질문 3: 학습모형 선택**
- 단원 특성을 분석하여 소주제별 모형 제안
- 옵션: 전체 발견중심형 / 전체 원리탐구형 / 혼합(소주제별 다름)

**질문 4: 출력 형식**
- DOCX (기본) / Markdown

**질문 5: 학습지(활동지) 연계 생성**
- 단원설계 완료 후 차시별 학습지도 함께 생성 (Recommended)
- 단원설계만 생성 (학습지는 나중에 `/cbci-worksheet` 로 별도 생성)

### Phase 3: 단원 설계 (팀즈 활용)

#### 3-1. 팀 생성 및 태스크 분배

TeamCreate로 팀을 생성하고, 소주제별로 에이전트를 병렬 스폰한다.

```
팀 구조:
- 팀 리더 (나): 교육과정 분석, 일반화 체계, 메인 스크립트, 검증
- st{N}-writer (소주제 수만큼): 소주제별 활동 콘텐츠 작성
```

#### 3-2. 팀 리더의 설계 작업

팀 리더는 다음을 직접 수행한다:

1. **단원 일반화 작성** (4대 원칙 적용)
   - "개념1 + 관계동사 + 개념2" 구조
   - 보편적, 추상적, 초월적
   - 인과/조건-결과의 복잡성
   - 완결된 문장

2. **소주제 일반화 분해**
   - 단원 일반화를 소주제 수만큼 분해
   - 각 소주제 일반화가 단원 일반화의 절(clause)과 1:1 대응
   - 소주제 순서가 단원 일반화의 논리 흐름과 일치

3. **핵심질문 및 논쟁적 질문**
   - 단원 핵심질문 1개 (단원 일반화에 도달하기 위한 고차적 사고)
   - 단원 논쟁적 질문 3개 (다양한 관점, 정답 없음)
   - 소주제별 핵심질문 1개씩

4. **평가 계획 (3차원)**
   - 1차원(이해): 개념/사실 확인 — 서답형
   - 2차원(적용): 맥락 적용/추론 — 서술형
   - 3차원(전이): 새 맥락 전이/실행 — 보고서/프로젝트

#### 3-3. 소주제 에이전트의 콘텐츠 작성

각 st{N}-writer 에이전트에게 다음 정보를 전달하여 콘텐츠 파일을 작성하게 한다:

**에이전트에 전달할 정보:**
- 소주제 일반화, 핵심질문, 렌즈 초점
- 차시별 탐구 단계 + 핵심 활동 개요
- 사용할 탐구기법 목록
- 활동 서술 형식 규칙 (▫◆★※ 기호)
- 이전 프로젝트 활동 예시 (형식 참고)
- framework_reference.md의 해당 부분

**에이전트 출력 파일 형식:**
```python
# _content_st{N}.py
ST{N}_GEN = "소주제 일반화..."
ST{N}_KEY_Q = "핵심 질문..."
ST{N}_INQUIRY_Q = "(사) ...\n(사) ...\n(사) ...\n(개) ...\n(개) ...\n(개) ...\n(논) ...\n(논) ...\n(논) ..."
ST{N}_CONCEPT_FOCUS = "개념1: ...\n개념2: ..."
ST{N}_LESSON{M} = """▫ [탐구기법] ...\n..."""
```

**에이전트 스폰 설정:**
```
Agent:
  name: st{N}-writer
  subagent_type: general-purpose
  team_name: {team_name}
  model: opus  (사용자가 별도 지정하지 않는 한 opus 사용)
  run_in_background: true
```

**탐구질문 규칙:**
- (사): 사실적 질문 3개 — 구체적 계산/관찰 기반
- (개): 개념적 질문 3개 — "왜", "어떻게" 등 원리 탐구
- (논): 논쟁적 질문 3개 — 다양한 관점, 정답 없음
- 소주제별 9개 × 소주제 수 = 총 탐구질문 수

**PYP 핵심개념 발문 (◆ 기호):**
- 각 차시에 2개씩
- 8개(형태, 기능, 원인, 변화, 연관성, 관점, 책임, 성찰) 고르게 분포

**탐구기법 매핑 (탐구 단계별):**
- 관계맺기: 의견기반전략, 경험기반전략, 토론기반전략
- 집중하기: 분류전략, 명명전략(프레이어 모델), 순위전략
- 조사하기: 체험, 동료멘토링, 시뮬레이션
- 조직+정리: 비교조직자(교차비교차트), 과정조직자(흐름도), 표상전략
- 일반화: 일반화 명료화/정당화/공유
- 전이: 근전이(소주제 초반), 원전이(마지막 소주제)
- 성찰: 3-2-1 전략, 학습저널 (매 소주제 마지막 필수)

### Phase 4: DOCX 생성

에이전트들의 콘텐츠 파일이 완성되면 메인 스크립트를 작성하고 실행한다.

1. **메인 스크립트 작성**: `generate_{단원명}.py`
   - `generate_docx_template.py`의 `build_unit_design()` 함수를 import
   - 소주제 콘텐츠 파일들을 import
   - 교육과정 분석, 소주제 구성, 탐구 질문, 평가 계획 데이터를 구성
   - `build_unit_design()` 호출

   ```python
   import sys
   sys.path.insert(0, r'C:\Users\sdm24\.claude\skills\cbci-unit-design\scripts')
   from generate_docx_template import build_unit_design
   from _content_st1 import *
   from _content_st2 import *
   # ...

   build_unit_design(
       unit_name='단원명',
       overview_data=[...],
       curriculum_data=[...],
       subtopics=[...],
       inquiry_rows=[...],
       eval_data=[...],
       subtopic_details=[...],
       output_file='단원명_단원설계_모형명.docx'
   )
   ```

2. **스크립트 실행**:
   ```bash
   cd "작업 디렉터리" && python generate_{단원명}.py
   ```

### Phase 5: 검증

생성된 DOCX의 품질을 검증한다:

```python
# 검증 항목
1. 테이블 수: 5 + (소주제 수 × 2) 개
2. 탐구질문: 소주제별 9개 (사실3 + 개념3 + 논쟁3) × 소주제 수
3. PYP 핵심개념 발문: 8개 모두 사용, 고른 분포
4. 일반화 체인: 소주제 → 단원 일반화 귀결 구조
5. 파일 정상 열림 확인
```

검증 스크립트를 작성하여 실행하고 결과를 사용자에게 보고한다.

### Phase 7: 학습지 연계 생성 (선택적)

Phase 2의 질문 5에서 "학습지도 함께 생성"을 선택한 경우에만 실행한다.

1. **결과파일 존재 확인**: Phase 3에서 생성된 결과파일들이 작업 디렉터리에 있는지 확인한다.
   - `result_task1_탐구기법매핑.md`
   - `result_task2_활동설계.md`
   - `result_task3_핵심개념체계.md`

   결과파일이 별도로 생성되지 않은 경우, Phase 3의 콘텐츠(`_content_st{N}.py`)와
   팀 리더가 작성한 일반화/질문 체계를 기반으로 3개 결과파일을 먼저 생성한다.

2. **`/cbci-worksheet` 스킬 호출**: Skill 도구로 연계 호출한다.
   ```
   Skill: cbci-worksheet
   args: "<작업 디렉터리> <단원명>"
   ```

   이때 학습지 스킬은 자체 Phase 0~6을 실행하여 `{단원명}_학습지_전체.docx`를 생성한다.

3. **최종 산출물 보고**: 단원설계 DOCX + 학습지 DOCX를 함께 보고한다.
   ```
   산출물:
   ├── {단원명}_단원설계_{모형}.docx    ← Phase 4에서 생성
   └── {단원명}_학습지_전체.docx       ← Phase 7에서 생성
   ```

## 학습모형별 탐구 단계 매핑

### 발견 중심형 (귀납적)
```
관계맺기 → 집중하기 → 조사하기 → 조직+정리 → 일반화+성찰
```

### 원리 탐구형 (연역적)
```
관계맺기 → 일반화 살펴보기 → 개념 탐색+정의 → 일반화 적용+평가
```

### 과정 구성형 (절차적)
```
관계맺기 → 기능/전략 분석 → 기능/전략 적용 → 일반화+평가
```

## 핵심 원칙

1. **귀납적 접근 우선**: 학생이 스스로 패턴을 발견하도록 설계. 교사가 먼저 결론을 제시하지 않는다.
2. **일반화 체인 정합성**: 소주제 일반화들이 단원 일반화로 귀결되는 논리 구조를 반드시 검증한다.
3. **탐구질문 균형**: 사실적/개념적/논쟁적 질문을 각 3개씩 균형 배치한다.
4. **PYP 발문 고른 분포**: 8개 핵심개념을 차시별로 고르게 배분한다.
5. **3차원 평가 연계**: 이해 → 적용 → 전이의 순차적 심화 평가를 설계한다.
6. **단원 특성 반영**: 모형 선택, 핵심개념 선정, 차시 배분을 단원 고유 특성에 맞게 조정한다.
